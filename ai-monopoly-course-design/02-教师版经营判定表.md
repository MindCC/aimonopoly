# 《AI 创新应用》模拟经营式综合答辩

## 教师版经营判定表

本表用于课堂过程性评价和期末综合答辩。建议教师每 1-2 次课安排一次经营回合，每次控制在 10-15 分钟。

## 一、核心指标

| 指标 | 教学含义 | 推荐判定依据 |
|---|---|---|
| 现金 | 项目资源和经营选择能力 | 课堂回答、阶段任务、策略收益、风险处罚 |
| 用户数 | 需求验证和市场接受度 | 用户访谈、试点、推广、真实反馈 |
| 产品力 | MVP 完整度和用户体验 | 原型、功能闭环、痛点匹配、迭代记录 |
| 技术力 | 课程知识应用能力 | Agent 架构、Prompt、Workflow、知识库、工具调用 |
| 声誉 | 表达、协作、口碑和可信度 | 展示质量、用户反馈、同伴建议、项目可信度 |
| 合规 | 安全、伦理、版权、隐私意识 | 数据来源、授权说明、边界设计、风险识别 |

## 二、课堂经营回合标准流程

1. 发布经营事件：结合本节课知识点设置一个现实情境。
2. 小组选择策略：每组从策略库中选择 1 个行动。
3. 角色解释：由对应角色说明选择理由。
4. 任务判定：通过回答问题、展示成果或提交材料判定成功程度。
5. 更新资源：教师公开记录资源变化。

## 三、课堂问题等级与奖励

| 等级 | 典型表现 | 奖励 |
|---|---|---:|
| 基础 | 能准确解释概念，如 Agent、Prompt、Workflow、MVP | 现金 +500 |
| 应用 | 能结合本组项目说明概念如何落地 | 现金 +1000 |
| 分析 | 能比较方案优劣，说明取舍和风险 | 现金 +1500 |
| 创新 | 能提出新方案、改进路径或跨工具组合 | 现金 +2000 或任一指标 +1 |

## 四、阶段成果判定

| 阶段 | 达标成果 | 推荐奖励 |
|---|---|---|
| 选题阶段 | 明确用户、问题、使用场景 | 现金 +1000，产品力 +1 |
| MVP 设计 | 有功能清单、核心流程、原型或结构图 | 现金 +1500，产品力 +1 |
| Agent 搭建 | 能运行基础 Bot 或 Agent | 技术力 +1，产品力 +1 |
| Workflow 实现 | 有多步骤流程、工具调用或条件分支 | 技术力 +2 |
| 用户验证 | 有访谈、测试、反馈或试点记录 | 用户 +30，声誉 +1 |
| 版本迭代 | 能展示优化前后变化 | 产品力 +1，技术力 +1 |
| 合规说明 | 说明数据来源、隐私边界、风险控制 | 合规 +2 |
| 期末演示 | 可完成核心功能演示 | 产品力 +2，声誉 +1 |

## 五、运营策略判定表

| 策略 | 成本 | 成功条件 | 成功效果 | 失败或风险 |
|---|---:|---|---|---|
| 用户访谈 | -500 | 提交访谈问题和结论 | 产品力 +1，声誉 +1 | 材料空泛则仅产品力 +0 |
| 竞品分析 | -800 | 说明 2 个竞品和差异化 | 产品力 +1，现金 +1000 | 差异化不清则无现金收益 |
| Prompt 优化 | -800 | 展示优化前后差异 | 技术力 +1 | 不能说明逻辑则无效 |
| 工作流重构 | -1500 | 展示流程图或关键节点 | 技术力 +2，产品力 +1 | 流程不可解释则只加技术力 +1 |
| 接入知识库 | -2000 | 说明数据来源和使用方式 | 技术力 +2 | 合规 <2 时，额外现金 -1000 |
| 快速 Demo | -1500 | 有可看或可运行 Demo | 产品力 +2 | 无法演示则产品力 +1 |
| 广告投放 | -2500 | 说明目标用户和投放理由 | 用户 +50 | 产品力 <3 时，声誉 -1 |
| 校园试点 | -1000 | 有明确试点场景 | 用户 +30，声誉 +1 | 场景虚泛则仅用户 +10 |
| 商业合作 | -1500 | 说明合作对象和收益逻辑 | 现金 +4000 | 商业逻辑不清则现金 +1000 |
| 免费试用 | -1000 | 产品力 >=2 | 用户 +60 | 现金 <3000 不可选 |
| 合规审查 | -1000 | 说明风险清单和防控措施 | 合规 +2 | 无 |
| 版本迭代 | -1500 | 展示前后变化 | 产品力 +1，技术力 +1 | 无明显变化则只加产品力 +0 |
| 降本优化 | -500 | 说明成本来源和优化方式 | 现金 +1000 | 技术力 <2 时无效 |
| 外包开发 | -3000 | 说明外包内容和验收标准 | 产品力 +2 | 技术力不增加 |

## 六、负面行为处理

| 情况 | 处理建议 |
|---|---|
| 迟交阶段任务 | 现金 -1000 |
| 小组成员长期缺席 | 现金 -1000，团队协作评价扣分 |
| 项目连续两轮无进展 | 产品力 -1 |
| 课堂经营解释明显不符合项目实际 | 本轮策略无效 |
| 使用未经说明的数据或材料 | 合规 -1 |
| 伪造用户反馈、截图或演示数据 | 现金 -3000，声誉 -2，合规 -2 |

## 七、期末资源换算建议

期末资源不直接决定最终成绩，但可以提供决战优势。

| 资源状态 | 期末影响 |
|---|---|
| 现金每 3000 元 | 可购买 1 张道具卡 |
| 现金排名第 1 | 期末总评可加 3 分，上限不超过 100 |
| 现金排名第 2 | 期末总评可加 2 分 |
| 现金排名第 3 | 期末总评可加 1 分 |
| 现金为负 | 期末总评扣 2 分 |
| 合规低于 2 | 期末必须回答 1 个合规问题 |
| 技术力低于 3 | 期末必须进行现场技术解释 |
| 产品力低于 3 | 期末必须补充 MVP 取舍说明 |

## 八、教师操作建议

- 每次只记录 1-2 个主要变化，避免课堂管理过重。
- 金额变化公开透明，指标变化说明原因。
- 鼓励学生解释决策，而不是追求最优答案。
- 风险卡和危机卡要服务课程目标，避免变成惩罚游戏。
- 期末评分仍以项目质量和答辩表现为主，经营资源作为过程性证据。
