# 卡牌库

## 使用方式

教师可将以下卡牌打印、剪裁，或复制到 PPT / 抽签工具中使用。建议期末每组抽 1-3 张卡牌，不宜过多。

## 一、机会卡

| 编号 | 卡牌名称 | 触发效果 |
|---:|---|---|
| O01 | 获得企业试点 | 产品力 >=4，现金 +5000；否则用户 +20 |
| O02 | 校园推广成功 | 用户 +50，声誉 +1 |
| O03 | 投资人关注 | 2 分钟讲清商业模式，成功则现金 +8000 |
| O04 | 行业数据开放 | 合规 >=3，技术力 +1，产品力 +1 |
| O05 | 用户自传播 | 声誉 >=4，用户 +80；否则用户 +30 |
| O06 | 教师推荐资源 | 可跳过一次危机卡 |
| O07 | 找到高频场景 | 产品力 +2 |
| O08 | 模型效果提升 | 技术力 +1，声誉 +1 |
| O09 | 社群试用反馈积极 | 用户 +40，产品力 +1 |
| O10 | 获得合作伙伴 | 说明合作价值，成功则现金 +4000 |
| O11 | 产品定位变清晰 | CEO 重新陈述定位，成功则产品力 +1，声誉 +1 |
| O12 | 技术方案被认可 | CTO 说明架构亮点，成功则技术力 +2 |
| O13 | 低成本获客 | CMO 说明渠道策略，成功则用户 +60 |
| O14 | 成本结构优化 | CFO 说明降本方案，成功则现金 +3000 |
| O15 | 合规准备充分 | 合规 +2，并免疫一次合规类处罚 |
| O16 | 用户提出关键建议 | 产品力 +1，并获得一次版本迭代机会 |

## 二、危机卡

| 编号 | 卡牌名称 | 处理要求 | 失败惩罚 |
|---:|---|---|---|
| C01 | AI 回答不稳定 | CTO 说明 Prompt 或 Workflow 优化方案 | 声誉 -1 |
| C02 | 用户隐私质疑 | CFO 说明数据处理方式和边界 | 现金 -3000，合规 -1 |
| C03 | 竞品上线同类功能 | CEO 说明差异化和取舍 | 用户 -30 |
| C04 | 模型调用成本上涨 | CFO 调整商业模式或成本结构 | 现金 -2000 |
| C05 | 用户不会用 | CMO 说明用户教育或产品简化方案 | 产品力 -1 |
| C06 | 演示现场失败 | CTO 现场补救或解释备用方案 | 声誉 -1 |
| C07 | 数据来源不清 | CFO 说明授权、来源和替代方案 | 合规 -2 |
| C08 | 功能太多太散 | CEO 重新定义 MVP | 产品力 -1 |
| C09 | 用户反馈不一致 | CMO 说明如何筛选有效反馈 | 用户 -20 |
| C10 | 团队分工混乱 | CEO 说明角色调整方案 | 声誉 -1 |
| C11 | 输出内容有事实错误 | CTO 说明校验机制 | 声誉 -1，技术力 -1 |
| C12 | 涉及版权素材 | CFO 说明版权处理方案 | 合规 -1 |
| C13 | 项目场景过宽 | CEO 收缩目标用户 | 产品力 -1 |
| C14 | 用户增长停滞 | CMO 设计新获客策略 | 用户 -20 |
| C15 | 商业模式不清 | CFO 说明收费对象和成本收益 | 现金 -2000 |
| C16 | 技术依赖单一平台 | CTO 说明替代方案或迁移路径 | 技术力 -1 |
